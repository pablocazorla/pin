Separating markup of style

En la mayoría de las ocaciones nos encontramos con el problema de que no siempre coincide el tipo de tag que queremos agregar con el estilo o modo en que se renderiza.

Ocurre que a veces no queremos que un tag h2 se vea como tal, sino más pequeño o más grande, por ejemplo.

Es por esto que Pin ofrece distintos modos de mostrar cualquier bloque de texto como un header, o un párrafo, etc.

Example - Different ways to render a h2 header

h2 tag with .h3 class

h2 tag without any .h2 or .title-2 class

h2 tag with .h2 class

p tag with .h2 class

Para dar el estilo correspondiente en cada caso, use las siguientes clases:

Style Class
Header 1 Use .h1 or .title-1 class.
Header 2 Use .h2 or .title-2 class.
Header 3 Use .h3 or .title-3 class.
Header 4 Use .h4 or .title-4 class.
Header 5 Use .h5 or .title-5 class.
Header 6 Use .h6 or .title-6 class.
Paragraph (p) Use .p class.

The '.textblock' class

En algunos casos es puede ser necesario no tener que agregar en cada caso la clase correspondiente para dar estilos a los headers o párrafos.

La '.textblock' class es especialmente útil para el caso de contenido de blogs, por ejemplo.

Para ese caso, basta wrap todo nuestro markup en un tag contenedor con la clase '.textblock'. En ese caso, cada tag se renderizará con los estilos correspondientes.
